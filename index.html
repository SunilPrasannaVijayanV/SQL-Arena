1. Database Server Installation

To run these queries locally on your machine, you need a Relational Database Management System (RDBMS). You have two major choices:

Option A: MySQL (Author's recommendation)

Best for lightweight, open-source setups. Excellent for learning command line and workbench tools.

  1. Download and install MySQL Community Server and MySQL Workbench.
  2. Complete installation, setting a password for the root user.
  3. Open MySQL Workbench and establish a local connection.

Option B: Microsoft SQL Server

Standard enterprise database system. The repository was built using MS SQL Server Management Studio (SSMS).

  1. Download and install SQL Server Express Edition (free version).
  2. Download and install SQL Server Management Studio (SSMS).
  3. Launch SSMS and connect using Windows Authentication mode.

2. Creating Database & Importing CSV Files

After installing your database, you must create a schema and import the datasets.

Method for MySQL Workbench:

  1. Run the SQL command to create a new database schema: 
    CREATE DATABASE hr_data;
CREATE DATABASE cars_info;
CREATE DATABASE call_centerdata;
  2. Right-click on the database schema in the sidebar, select Table Data Import Wizard.
  3. Choose the corresponding CSV file (e.g. Analyzing Employee Trends.csv) and name the destination table (e.g. hrdata).
  4. Define data types: ensure columns like age, job_satisfaction, selling_price are read as INT/DOUBLE.
  5. Click import to populate the database tables!

Method for SQL Server (SSMS):

  1. Right-click on Databases -> New Database, name it (e.g. hr_data).
  2. Right-click the newly created database -> Tasks -> Import Flat File...
  3. Browse for the CSV file and define column schemas. Pay close attention to stripping text prefixes if importing numerical metrics (e.g. mileage, engine) or change their types to Varchar and handle them later.
  4. Follow the prompts to finalize the import.

3. Executing SQL Files

Once the tables are imported successfully, you can copy-paste the SQL script queries from our workspace into your SQL runner to analyze the database tables.

Steps to run queries:

  1. Open a new query window in MySQL Workbench or SSMS.
  2. Ensure you target the correct schema using: 
    USE hr_data; -- For Employee Trends
USE cars_info; -- For Automotive Trends
USE call_centerdata; -- For Call Center Data
  3. Copy individual queries from the .sql files in your workspace, paste them into the query editor, select a query, and click Execute (⚡) to run it.

4. MySQL vs SQL Server Dialect Differences

SQL standards exist, but MySQL, SQL Server, and SQLite use slightly different dialects. If you run into issues, check these syntax variations:
